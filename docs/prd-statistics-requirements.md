## 1. 背景

### 1.1 当前问题

当前统计功能存在严重的数据不准确问题：

**实际场景**（2025-10-31）:
```
用户在电脑前工作 5 小时
期间多次推迟休息
每次工作会话都有推迟操作
```

**显示的统计数据**:
```
工作会话：4 次              ❌ 实际应该更多
最长连续工作：30 分钟       ❌ 推迟时长未计入
完成休息次数：24 次         ⚠️  包含了推迟会话
推迟次数：0 次 (0%)         ❌ 实际推迟了很多次
```

### 1.2 根本原因

1. **概念混淆**：推迟会话被当作休息会话记录
2. **数据丢失**：推迟信息没有关联到工作会话
3. **统计错误**：工作时长没有包含推迟时长

## 2. 核心概念重新定义

### 2.1 三种会话类型

#### Work Session（工作会话）
**定义**: 用户连续使用电脑的工作时段

**生命周期**:
```
开始 → 工作计时 → 到达设定时长/用户主动休息 → 提示休息
     ↓
     可能推迟（1-N次）
     ↓
     最终休息 → 结束
```

**属性**:
- `planned_duration`: 标准时长（如 30 分钟）
- `actual_work_duration`: 工作时长（实际连续工作，包含推迟）
- `postpone_count`: 推迟次数
- `postpone_total_duration`: 推迟总时长
- `break_completed`: 是否完成休息

**计算规则**:
```
工作时长 = 标准时长 + 推迟总时长
例如：30 分钟 + 8 分钟推迟 = 38 分钟工作时长
```

#### Postpone Session（推迟会话）
**定义**: 用户推迟休息的时段

**生命周期**:
```
休息提示 → 用户点击推迟 → 推迟倒计时 → 再次提示休息
```

**属性**:
- 推迟时长（1分钟、2分钟、5分钟）
- 关联的 Work Session

**重要**: 推迟时长计入对应 Work Session 的"工作时长"

#### Break Session（休息会话）
**定义**: 用户真正完成的休息时段

**生命周期**:
```
开始休息 → 休息倒计时 → 完成 → 开始新的 Work Session
```

**可能的中断场景**:
```
开始休息 → 休息倒计时中...
  ↓
  系统事件发生（合盖/休眠/屏保）
  ↓
  系统唤醒 → 直接开始新的 Work Session
  ↓
  Break Session 被中断，标记为 interrupted
```

**属性**:
- 休息时长（如 3 分钟）
- 关联的 Work Session
- 状态：
  - `completed`: 正常完成
  - `interrupted`: 被系统事件中断

**重要**: 只有 `status = completed` 的 Break Session 才计入"完成休息次数"

## 3. 用户场景示例

### 3.1 完整的工作-休息周期

**时间线**:
```
10:00 - 10:30   工作 30 分钟
10:30           提示休息 → 用户点击"推迟 5 分钟"
10:35           提示休息 → 用户点击"推迟 2 分钟"
10:37           提示休息 → 用户点击"推迟 1 分钟"
10:38           提示休息 → 用户真正开始休息
10:38 - 10:41   休息 3 分钟（完成）
10:41           开始新的工作会话
```

**会话记录**:
```
Work Session #1:
  - 标准时长: 30 分钟
  - 工作时长: 38 分钟 (30 + 8)
  - 推迟次数: 3 次
  - 推迟总时长: 8 分钟
  - 完成休息: 是

  包含:
  ├── Postpone Session #1: 5 分钟
  ├── Postpone Session #2: 2 分钟
  ├── Postpone Session #3: 1 分钟
  └── Break Session #1: 3 分钟 (completed)
```

### 3.2 用户主动触发休息

**时间线**:
```
11:00 - 11:15   工作 15 分钟
11:15           用户点击菜单"现在休息"
11:15 - 11:18   休息 3 分钟（完成）
11:18           开始新的工作会话
```

**会话记录**:
```
Work Session #2:
  - 标准时长: 30 分钟
  - 工作时长: 15 分钟（用户主动结束）
  - 推迟次数: 0 次
  - 推迟总时长: 0 分钟
  - 完成休息: 是

  包含:
  └── Break Session #2: 3 分钟 (completed)
```

### 3.3 休息期间系统唤醒场景

**时间线**:
```
14:00 - 14:30   工作 30 分钟
14:30           提示休息 → 用户开始休息
14:30 - 14:31   休息 1 分钟后，用户合盖去开会
15:30           用户回来，打开电脑（系统唤醒）
15:30           应用重置，开始新的工作会话
```

**会话记录**:
```
Work Session #3:
  - 标准时长: 30 分钟
  - 工作时长: 30 分钟
  - 推迟次数: 0 次
  - 推迟总时长: 0 分钟
  - 完成休息: 否（系统事件中断）

  包含:
  └── Break Session #3: 1 分钟 (interrupted, 被系统唤醒中断)
```

### 3.4 工作期间系统唤醒场景

**时间线**:
```
16:00 - 16:25   工作 25 分钟
16:25           用户合盖去开会（未到30分钟）
17:00           用户回来，打开电脑（系统唤醒）
17:00           应用重置，开始新的工作会话
```

**会话记录**:
```
Work Session #4:
  - 标准时长: 30 分钟
  - 工作时长: 25 分钟（系统唤醒中断）
  - 推迟次数: 0 次
  - 推迟总时长: 0 分钟
  - 完成休息: 否（系统事件中断）
```

## 4. 统计报表需求

### 4.1 今日用眼状况

```
📊 今日用眼状况

工作会话: 8 次
最长工作时长: 40 分钟
完成休息次数: 7 次
推迟次数: 15 次 (25%)
```

**字段定义**:

| 显示名称 | 计算规则 | 说明 |
|---------|---------|------|
| 工作会话 | COUNT(Work Session) | 今天总共几个工作会话 |
| 最长工作时长 | MAX(actual_work_duration) | 最长的连续工作时长（含推迟） |
| 完成休息次数 | COUNT(break_completed = true) | 完成休息的工作会话数量 |
| 推迟次数 | SUM(postpone_count) | 总推迟次数 |
| 推迟率 | SUM(postpone_count) / COUNT(Work Session) | 平均每个会话推迟几次 |

### 4.2 今日最长工作时段

```
⏰ 今日最长工作时段

最长连续: 40 分钟
```

**计算规则**:
- 找出今天所有 Work Session 的 `actual_work_duration`
- 取最大值

### 4.3 本周统计

```
📈 本周统计

总工作时长: 122 小时
健康天数: 6/7 天
周推迟率: 18%
```

**字段定义**:

| 显示名称 | 计算规则 | 说明 |
|---------|---------|------|
| 总工作时长 | SUM(actual_work_duration) | 本周所有工作时长之和 |
| 健康天数 | COUNT(每日推迟率 < 30%) | 推迟率低于30%的天数 |
| 周推迟率 | SUM(postpone_count) / COUNT(Work Session) | 本周平均推迟率 |

## 5. 需求变更对比

### 5.1 概念变更

| 项目 | v1.0/v1.1 | v1.2（新需求） |
|------|-----------|---------------|
| Work Session 定义 | 只包含工作时间 | 包含工作 + 推迟时间 |
| Postpone 记录 | 没有明确记录 | 明确记录并关联到 Work Session |
| Break Session | 包含推迟和真实休息 | 仅包含真正完成的休息 |
| 工作时长 | 固定30分钟 | 实际时长（可能 < 30 或 > 30） |

### 5.2 统计指标变更

| 指标 | v1.0/v1.1 | v1.2（新需求） | 变更原因 |
|------|-----------|---------------|---------|
| 工作会话 | ？ | COUNT(Work Session) | 需要准确统计 |
| 最长连续工作 | 30分钟（固定） | MAX(工作时长 + 推迟) | 推迟应计入连续工作 |
| 完成休息次数 | Break Session 数量 | 完成休息的 Work Session 数 | 推迟不算完成休息 |
| 推迟次数 | 0（未记录） | SUM(postpone_count) | 新增统计 |

## 6. 验收标准

### 6.1 数据准确性

场景测试：
```
1. 工作 30 分钟 → 推迟 5 分钟 → 推迟 2 分钟 → 完成休息

   预期统计:
   - 工作会话: +1
   - 最长工作时长: 37 分钟（30+5+2）
   - 完成休息: +1
   - 推迟次数: +2
```

### 6.2 边界情况

1. **用户主动休息**（工作不足30分钟）
   - 工作时长应为实际时长（如 15 分钟）
   - 推迟次数为 0

2. **系统唤醒中断**
   - 工作时长为中断前的实际时长
   - `break_completed = false`

3. **推迟到达上限**（10分钟）
   - 推迟总时长: 10 分钟
   - 推迟次数: 根据实际点击次数
