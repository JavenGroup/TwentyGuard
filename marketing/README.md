# Marketing

This directory is the jstudio working area for TwentyGuard positioning, launch
planning, public copy, channel research, and visual assets.

The marketing direction should stay aligned with the product itself:

- Native macOS menu bar app.
- Local-first and private.
- Built around the 20-20-20 eye-rest rule.
- Calm in appearance, strict in behavior.
- Differentiated by postpone limits, health statistics, and optional night screen lock.

## Brand

Public brand:

```text
TwentyGuard
```

Current tagline:

```text
Strict 20-20-20 breaks for macOS.
```

GitHub repository:

```text
https://github.com/JavenGroup/TwentyGuard
```

Before broader public launch, still run final checks for App Store Connect,
domains, and basic trademark risk.

## Current Launch Story

```text
Most break reminders are easy to ignore. TwentyGuard is for people who need the
break to actually happen.
```

Use this idea as the center of public messaging. The product should feel calm,
native, and lightweight, but the differentiation is enforcement: full-screen
breaks, cumulative postpone limits, health rhythm feedback, and optional night
screen lock.

## Directory Map

- `positioning.md` - audience, message, differentiation, and one-line descriptions.
- `naming-research.md` - current name recommendation and collision notes.
- `launch-plan.md` - launch sequence, channel order, and operating cadence.
- `launch-readiness.md` - status checklist for what is done, missing, or needs verification.
- `copy-bank.md` - reusable public copy in English and Chinese.
- `assets-shotlist.md` - screenshots, demo video/GIF, and visual asset requirements.
- `channels/` - channel-specific posting notes and draft prompts.
- `evidence/` - dated evidence for naming, domain, App Store, trademark, and search checks.
- `assets/` - screenshots, short videos, and other launch materials.
- `icons/` - app icon, menu bar glyph, icon previews, and brand usage notes.

## Operating Rules

- Keep public-facing copy factual and modest.
- Do not make medical claims beyond general eye-strain prevention language.
- Emphasize that data is local unless the app behavior changes.
- Validate channel rules before posting.
- Keep `20-20-20` in subtitles and SEO even if the main brand changes.
- Separate confirmed facts from assumptions; record verification dates in `evidence/`.
- Treat jstudio work as marketing work by default. Keep code inspection limited to
  accuracy checks for user-facing claims.
